📚 RAPOS Frontend - JavaScript API Reference

Complete documentation for utility libraries and available functions.

🎯 Modal.js - Dialog Component

Reusable modal component for alerts, confirmations, and loading states.

Global Instance

The modal is automatically initialized. Access it via the global modal object.

modal.alert(title, message)

Show an alert dialog with OK button.

Parameters:
• title (string): Dialog title
• message (string): Message to display
Returns:
Promise (resolves when OK clicked)
Example: 
modal.alert('Success', 'Your profile has been updated!');

modal.confirm(title, message)

Show a confirmation dialog with Cancel/Confirm buttons.

Parameters:
• title (string): Dialog title
• message (string): Message to display
Returns:
Promise (true if confirmed, false if cancelled)
Example: 
const confirmed = await modal.confirm('Delete Account', 'This action cannot be undone.');
if (confirmed) {
    console.log('Account deleted');
}

modal.loading(title)

Show a loading dialog with spinner.

Parameters:
• title (string, optional): Loading message (default: "Loading...")
Example: 
modal.loading('Processing your request...');
// Do async operation
setTimeout(() => {
    modal.close();
}, 3000);

modal.close()

Close the currently active modal.

Example: 
modal.close();

♿ Accessibility.js - Validation & Keyboard

Utilities for form validation, keyboard navigation, and password strength.

KeyboardAccessibility

Handles keyboard navigation (Tab, Enter, Escape).

KeyboardAccessibility.init()

Initialize keyboard accessibility features. Auto-called on page load.

Features:
  • Tab navigation through all interactive elements
  • Enter key to submit forms
  • Escape key to close modals

FormValidator

Static methods for form field validation.

FormValidator.validateEmail(email)

Validate email format.

Parameters:
• email (string): Email to validate
Returns:
boolean (true if valid)
Example: 
if (FormValidator.validateEmail('user@example.com')) {
    console.log('Valid email');
}

FormValidator.validatePassword(password)

Validate password strength (8+ chars, uppercase, lowercase, number, symbol).

Returns:
boolean (true if meets requirements)

FormValidator.validateField(field)

Validate a single form field and show error messages.

Parameters:
• field (HTMLElement): Input field to validate
Returns:
boolean (true if valid)

FormValidator.setupRealTimeValidation(form)

Enable real-time validation for a form.

Parameters:
• form (HTMLElement): Form element
Example: 
const form = document.querySelector('form');
FormValidator.setupRealTimeValidation(form);

PasswordStrength

Display password strength indicator while typing.

PasswordStrength.init()

Initialize password strength indicators for all password fields. Auto-called on page load.

Features:
  • Real-time strength feedback (Very Weak → Very Strong)
  • Color-coded strength levels
  • Works on inputs with name="password" or id="password"

📱 Responsive.js - Device Detection & Tooltips

Utilities for responsive design and tooltips.

ResponsiveUtils

Device detection and responsive layout utilities.

Method Returns Description
ResponsiveUtils.isMobile() boolean true if viewport width ≤ 768px
ResponsiveUtils.isTablet() boolean true if 768px < width ≤ 1024px
ResponsiveUtils.isDesktop() boolean true if width > 1024px

ResponsiveUtils.onResize(callback)

Execute callback when window is resized (debounced).

Parameters:
• callback (function): Function to execute on resize
Example: 
ResponsiveUtils.onResize(() => {
    if (ResponsiveUtils.isMobile()) {
        // Adjust layout for mobile
    }
});

Tooltip

Static methods for adding hover tooltips to elements.

Tooltip.add(element, text, position)

Add a tooltip to an element.

Parameters:
• element (HTMLElement): Element to add tooltip to
• text (string): Tooltip text
• position (string, optional): 'top' or 'bottom' (default: 'top')
Example: 
const btn = document.getElementById('myButton');
Tooltip.add(btn, 'Click to save your changes', 'top');

🔧 Global Functions

showToast('Profile updated successfully!', 3000);

Parameters:

📝 Common Implementation Patterns

Add Validation to a Form

const form = document.querySelector('#myForm');
FormValidator.setupRealTimeValidation(form);

form.addEventListener('submit', function(e) {
    e.preventDefault();
    let isValid = true;
    form.querySelectorAll('input').forEach(input => {
        if (!FormValidator.validateField(input)) {
            isValid = false;
        }
    });
    if (isValid) {
        modal.loading('Submitting...');
        // Process form
    }
});

Add Tooltip to Multiple Elements

document.querySelectorAll('[data-tooltip]').forEach(el => {
    const text = el.getAttribute('data-tooltip');
    Tooltip.add(el, text);
});

Responsive Layout Change

ResponsiveUtils.onResize(() => {
    const grid = document.querySelector('.features-grid');
    if (ResponsiveUtils.isMobile()) {
        grid.style.gridTemplateColumns = '1fr';
    } else if (ResponsiveUtils.isTablet()) {
        grid.style.gridTemplateColumns = 'repeat(2, 1fr)';
    } else {
        grid.style.gridTemplateColumns = 'repeat(3, 1fr)';
    }
});

Dark Mode Toggle Implementation

const toggle = document.getElementById('darkModeToggle');
toggle.addEventListener('change', function() {
    localStorage.setItem('darkMode', this.checked);
    setDarkMode(this.checked);
    showToast(this.checked ? 'Dark mode ON' : 'Dark mode OFF');
});

⚡ Performance Tips

  • All utilities are lazy-loaded only when needed
  • Use event delegation for dynamic content
  • Debounce resize events (already done in ResponsiveUtils)
  • Cache frequently accessed DOM elements
  • Use CSS classes instead of inline styles when possible
  • Minimize localStorage reads/writes

🐛 Troubleshooting

Issue: Modal not appearing

Solution:

  • Ensure modal.js is loaded before use
  • Check that #modal element exists in DOM
  • Verify z-index is not being overridden
Issue: Form validation not working

Solution:

  • Call FormValidator.setupRealTimeValidation() on form
  • Check input names/ids match validation rules
  • Ensure accessibility.js is loaded
Issue: Responsive design not working on mobile

Solution:

  • Add viewport meta tag: <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Clear browser cache
  • Test in actual mobile or DevTools device emulation
