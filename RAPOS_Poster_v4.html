Introduction

Retail investors lack access to institutional-grade analytics. RAPOS democratises quantitative finance with a production-grade AI terminal — fully deployed and live on Vercel.

3.8B+
Portfolio volume managed · 12K+ institutional nodes active worldwide
Problem Statement
  • Predictive analytics locked on Bloomberg/Refinitiv (high cost)
  • Manual rebalancing without any quantitative guidance
  • Fragmented data — prices, news, and portfolio in silos
  • Reactive decisions made after market trends already move
  • No unified view for retail investors across asset classes
Research Question

"Can an LSTM trained on 60-day OHLCV sequences predict next-day closing prices and drive an automated, risk-adjusted rebalancing system at production scale?"

Why LSTM for Finance?
✅ Handles sequential time-series data natively
✅ Memory gates capture long-term dependencies
✅ Outperforms ARIMA on non-linear patterns
✅ Scales to 500+ equities simultaneously
Methodology — LSTM Architecture
INPUT LAYER
60-day window · OHLCV + 6 technical indicators · 11 features per timestep
LSTM — 128 units
return_sequences = True · learns long-range temporal patterns in price series
LSTM — 64 units
Dropout = 0.2 · prevents overfitting · captures high-level market context
Dense — 32 neurons · ReLU activation
Fully connected layer · non-linear feature combination
OUTPUT — Predicted Close · AI Risk Score (1–10) · Sentiment Signal
Optimizer: Adam · Loss: MSE · Epochs: 50 · Keras/TensorFlow
